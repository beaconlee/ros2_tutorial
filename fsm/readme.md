### node 分开实现

在进行实际开发时，有一种策略是将 node 的实现和其它代码的实现分开，这种策略的好处是，可以在不同的主程序中复用同一个节点实现，只需通过不同的启动配置（launch文件）进行管理。



### QoS 的策略

在 ROS2 中，为了确保通信能够成功，发布者和订阅者的QoS设置必须兼容。

为什么会通信失败：
1. QoS 不兼容：
  publisher 的 best effort 不提供消息的重传和确认机制
  subscribe 的 reliable 策略需要确认每条消息的接收情况，并在丢失时请求重传

2. 通信协议不匹配：
  发布者发布消息后，不会等待确认，也不会重传丢失消息
  订阅者希望每条消息都被成功接收，如果没有接收到消息，会请求重传，但发布者不支持重传机制



### h 和 hpp 文件的区别

在 ros2 中 h 头文件和 hpp 头文件的区别：
1. 代码生成方式：

  .hpp 文件：这些文件是通过ROS2的 rosidl_generator_cpp 生成的C++头文件，使用的是ROS2的现代 C++ API。
  .h 文件：  这些文件是通过ROS2的 rosidl_generator_c   生成的C头文件，使用的是 C 语言API。


2. 语言特性：

  .hpp 文件：包含了C++的语言特性，如类、智能指针、标准模板库（STL）等，更符合现代C++编程习惯。
  .h 文件：  仅使用C语言的特性，没有C++的类和智能指针等特性。


3. API稳定性：

  .hpp 文件：提供更高级别、更稳定的API，推荐用于C++开发。
  .h 文件：  主要用于需要与纯C语言代码兼容的场景或需要极简依赖的场合。

4. 适用场景：

  .hpp 文件：大多数情况下，尤其是C++开发，应该使用.hpp文件，以利用C++的特性和更高级的API。
  .h 文件：  在需要与C语言代码进行互操作，或者在嵌入式系统中需要极简依赖时，可以考虑使用.h文件。



> 为什么在ROS2中推荐使用 .hpp 文件？
  C++特性：现代C++特性使得代码更简洁、安全和高效。例如，使用智能指针（如std::shared_ptr）来管理内存，而不是手动管理。
  标准接口：.hpp文件提供了标准的C++接口，与ROS2其他C++库和工具更好地集成。
  社区惯例：ROS2社区和大多数教程、示例代码中也普遍使用.hpp文件，遵循社区惯例有助于代码的可读性和维护性。
